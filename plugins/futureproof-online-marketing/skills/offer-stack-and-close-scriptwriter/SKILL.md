## Step 3: Do the Work

Execute the following six-phase methodology sequentially. Each phase produces a discrete, named script block.

### Phase 1: Power Offer Statement (Erway Method)

Compress the entire offer into a razor-sharp 1–2 sentence Power Offer Statement following Erway's high-ticket formula:

> **[Specific outcome] for [specific ICA] in [specific timeframe] without [primary objection/fear] — even if [secondary objection/excuse].**

Draft 3 variants ranked by specificity and emotional charge. The winning variant becomes the throughline for every subsequent script block.

### Phase 2: Value Stack Architecture (Brunson Method)

Build the value stack as an ordered sequence of stack elements. For each element, script:

| Stack Element | Script Component |
|---|---|
| **Core Offer** | Name, description framed as outcome (not feature), assigned perceived value |
| **Bonus 1 — Speed** | Accelerator asset that collapses the timeline; perceived value |
| **Bonus 2 — Ease** | Done-for-you or template asset that removes friction; perceived value |
| **Bonus 3 — Access** | Community, coaching, or support asset that eliminates isolation; perceived value |
| **Bonus 4 — Proof** | Case study vault, swipe file, or accountability mechanism; perceived value |
| **Bonus 5+ (optional)** | ICA-specific situational bonuses surfaced from context |

For each element, write:
- A **headline** (benefit-driven, 8 words max)
- A **3–4 sentence value justification** spoken in ICA language
- A **perceived value anchor** (with brief rationale so it is defensible, not arbitrary)

Script the **running total slide narration** — the cumulative value buildup the presenter reads aloud as each element stacks.

### Phase 3: Price Reveal & Contrast Script

Script the price reveal sequence using Brunson's contrast architecture:

1. **Total Value Recap** — restate the cumulative perceived value ("If you purchased each of these individually, you'd invest $X…")
2. **Price Anchor Drop #1** — "But you're not going to pay $X…"
3. **Price Anchor Drop #2** — "You're not even going to pay $Y…"
4. **Actual Price Reveal** — "Your total investment today is just $Z."
5. **Price Justification Frame** — contextualise $Z against a relatable daily/weekly cost or the cost of inaction
6. **Payment Plan Option** (if applicable) — script the plan introduction with no apology framing

Include stage directions for vocal pacing, pauses, and slide transitions.

### Phase 4: Risk Reversal & Guarantee Script

Script the guarantee section using one of the following models (recommend the optimal model based on offer type and ICA risk sensitivity):

- **Unconditional Money-Back Guarantee** — full refund, no questions, stated timeframe
- **Conditional Results Guarantee** — "Do the work, show us proof, and if you don't get [outcome], we'll [refund / continue coaching / pay for alternative]"
- **Double Guarantee (Brunson)** — combine a short-window unconditional + long-window conditional
- **Anti-Guarantee (Fladlien)** — for high-ticket where a guarantee would diminish perceived exclusivity; script a "we guarantee nothing except…" authority frame

For each, write the verbatim script the presenter reads, including the reframe that turns the guarantee into a confidence signal rather than a safety net.

### Phase 5: Scarcity & Urgency Layer

Script authentic scarcity/urgency elements. Select and script from:

| Type | Script Template |
|---|---|
| **Quantity scarcity** | Limited seats, cohort caps, inventory limits (must be real) |
| **Time urgency** | Deadline, cart close, fast-action window |
| **Bonus scarcity** | Bonuses available only during live event / first X buyers |
| **Price urgency** | Introductory pricing, price increase after deadline |
| **Situational urgency** | Market window, seasonal relevance, regulatory change |

Script each selected element as a discrete paragraph the presenter can deliver. Include the **reason why** — Fladlien's principle that scarcity without a credible reason triggers scepticism.

### Phase 6: Close Sequence & CTA

Script **four close variations** so the user can A/B test or layer them within a single presentation:

#### Close A — Brunson Stack & Close (Standard)
The classic "you're getting [recap stack], all for just [price], click the button below" direct close. Script verbatim.

#### Close B — Fladlien Gratitude Close
Transition from selling to genuine gratitude. Script: thank the audience for their time, restate belief in them, then make the offer from a posture of service. The CTA emerges from gratitude, not pressure.

#### Close C — Fladlien Interactivity Close
Script 2–3 micro-engagement prompts ("Type YES in the chat if…", "Raise your hand if…") that lead logically into the CTA. Each prompt escalates commitment.

#### Close D — Fladlien User's Manual Close
Script a "here's exactly what happens after you click the button" walkthrough — step 1 you'll receive…, step 2 you'll schedule…, step 3 you'll start… — that makes the post-purchase experience tangible and reduces fear of the unknown.

**CTA Selection Framework**: Based on price point, ICA sophistication, and funnel type, recommend the optimal CTA:
- **Buy Now** → offers under $2,000, self-liquidating funnels, impulse-friendly ICAs
- **Book a Call** → offers $2,000–$15,000, complex deliverables, ICAs who need human reassurance
- **Apply** → offers $15,000+, capacity-constrained, ICAs where exclusivity drives desire

Script the CTA button language, the verbal instruction, and the post-CTA reassurance line.

### Phase 7: Objection Handling Scripts

Identify the top 5 objections for this specific ICA and offer (use FutureProof context if prior objection data exists). For each objection, script:

1. **Acknowledge** — validate the concern without agreeing with the premise
2. **Reframe** — shift the frame from cost/risk to cost of inaction or hidden upside
3. **Evidence** — one proof element (testimonial excerpt, case study micro-story, statistic)
4. **Re-close** — transition back to the CTA

Script these as:
- **Pre-emptive insertions** — lines woven into the stack/close that neutralise the objection before it forms (Fladlien's Built-In Persuasion Close)
- **Live Q&A / chat responses** — standalone paragraphs the presenter can deliver if the objection surfaces in real time
- **Follow-up email / DM scripts** — for post-presentation objection handling sequences

## Step 4: Deliver Output

Compile and deliver the following named documents:

### Document 1: Offer Stack & Close Master Script
A single, presentation-ready script containing:
- Power Offer Statement (winning variant, plus 2 alternates in an appendix)
- Value Stack slide-by-slide narration (with stage directions)
- Price Reveal sequence
- Risk Reversal / Guarantee section
- Scarcity / Urgency block
- All four Close variations (clearly labelled so the user can mix and match)
- CTA block with recommended CTA model and rationale

Format: numbered slides/sections with **[SLIDE]** markers, *[PAUSE]* directions, and `[CLICK]` cues.

### Document 2: Objection Handling Playbook
- Top 5 objections with pre-emptive, live, and follow-up scripts
- Organised in a table: Objection | Pre-emptive Line | Live Response | Follow-up Script

### Document 3: Value Stack Summary Slide (Copy-Ready)
- A single-slide text layout the user can paste directly into Keynote/Google Slides/PowerPoint:
  - Each stack element with headline and perceived value
  - Total value line
  - Actual price line
  - Savings line

### Document 4: CTA Decision Matrix
- One-page matrix mapping price point × ICA type × funnel vehicle → recommended CTA model, with brief rationale per cell

Include a **Performance Benchmarks** note: industry-standard conversion ranges for the user's funnel type (e.g., live webinar 5–15%, automated webinar 2–8%, VSL 1–5%, application funnel 20–40% of applicants) so the user can contextualise results.
