## Step 2: Get Input

Ask the user to provide the following (accept partial input and probe for gaps):

1. **Core offer**: What is the primary product, programme, or service being sold? What is the price point (or intended price point)?
2. **ICA profile**: Who is the ideal customer? What is their current painful state, desired future state, and the language they use to describe both?
3. **Delivery context**: Where will this offer be presented? (webinar, sales page, VSL, live stage, application funnel, DM close)
4. **Existing assets**: Are there any current bonuses, guarantees, or offer elements already drafted?
5. **Competitive landscape**: What alternatives does the ICA perceive they have? What price anchors exist in the market?
6. **Conversion objective**: What is the primary metric — front-end conversion rate, average order value, refund reduction, or ascension to a higher tier?

## Step 3: Architect the Core Offer Value Thesis

Before stacking bonuses, establish the foundational value architecture:

### 3A: Transformation Mapping

Define the offer's value through the **Transformation Delta**:

| Dimension | ICA Current State | ICA Desired State | Offer Mechanism |
|---|---|---|---|
| Financial | e.g., inconsistent $5K months | e.g., predictable $25K months | e.g., client acquisition system |
| Temporal | e.g., 60-hour weeks | e.g., 25-hour weeks | e.g., automation workflows |
| Emotional | e.g., anxiety, impostor syndrome | e.g., confidence, authority | e.g., community + coaching |
| Status | e.g., invisible in market | e.g., recognised category leader | e.g., positioning framework |

### 3B: Vehicle, Internal, External Belief Identification (Brunson Framework)

Identify the three belief categories the offer must overcome:

1. **Vehicle belief**: "Will *this specific method* work?" — Map the unique mechanism and why it is different/better
2. **Internal belief**: "Can *I* make it work?" — Map enablers that remove personal capability objections
3. **External belief**: "Will *external factors* prevent my success?" — Map environmental de-risking elements

### 3C: Standalone Value Assignment

For the core offer itself (before any bonuses), assign a **defensible standalone value** using one of three valuation methods:

- **Cost-to-replicate**: What would it cost the ICA to assemble this outcome themselves? (consultants, tools, time)
- **Revenue attribution**: What revenue or savings will this generate within 12 months for a median ICA?
- **Market comparable**: What do competitors or adjacent solutions charge for equivalent transformation?

Document the chosen method and the resulting core offer value figure.

## Step 4: Engineer the Bonus Stack

Design each bonus as a strategic conversion lever — not filler. Every bonus must serve one of four functions:

### Bonus Architecture Matrix

| Bonus | Strategic Function | Addresses Which Belief? | Standalone Value | Valuation Method |
|---|---|---|---|---|
| Bonus 1 | **Speed** — accelerates time-to-result | Vehicle / Internal | $X,XXX | Cost-to-replicate |
| Bonus 2 | **Completeness** — fills a gap the ICA anticipates | External | $X,XXX | Market comparable |
| Bonus 3 | **Objection killer** — neutralises #1 purchase objection | Internal / External | $X,XXX | Revenue attribution |
| Bonus 4 | **Desire amplifier** — adds aspirational upside beyond core promise | Vehicle | $X,XXX | Market comparable |
| Bonus 5 (if applicable) | **Scarcity/exclusivity** — available only in this offer configuration | Any | $X,XXX | Cost-to-replicate |

**Bonus Design Rules** (Fladlien methodology):

1. Each bonus must be nameable as a standalone product (proper noun, not generic description)
2. Each bonus must have a credible standalone price the ICA would believe if sold separately
3. Bonuses should cascade logically — each one makes the previous more valuable
4. At least one bonus should address the ICA's *unspoken* objection (the one they will not voice on a sales call)
5. No bonus should cannibalise or contradict the core offer's positioning

Apply Erway's Power Offer principle: if the bonuses alone would justify the price, the core offer becomes perceived as "free."

## Step 5: Design the Risk Reversal Architecture

Engineer the guarantee structure using a tiered approach:

### Guarantee Tier Assessment

| Guarantee Type | Description | Best When | Risk to Seller |
|---|---|---|---|
| **Unconditional** | Full refund, no questions, 30/60/90 days | High trust needed, low fulfilment cost | High |
| **Conditional** | Full refund if ICA completes X steps and does not achieve Y result | Coaching/programme offers | Medium |
| **Performance** | Seller guarantees a specific measurable outcome or continues working for free | High-ticket, high-confidence offers | Medium-High |
| **Anti-guarantee** | Explicitly no refund — framed as commitment filter | Premium/exclusive, identity-driven offers | Low |
| **Double guarantee** | Conditional + unconditional layered (Brunson "dual guarantee") | Webinar offers, mid-ticket | Medium |

Recommend the optimal guarantee tier based on:
- Price point and ICA risk sensitivity
- Seller's fulfilment confidence and historical completion rates
- Delivery context (refund dynamics differ for webinar vs. application-close offers)
- Any FutureProof experiment data on prior guarantee performance

Draft the guarantee in precise, presentation-ready language — including the specific conditions, timeframe, and claim process.

## Step 6: Construct the Stack Slide & Value Summary

Build the cumulative **Stack Slide** — the single visual artefact that presents total value against the ask price.

### Value-to-Price Ratio Assessment

Calculate and evaluate:
- **Value-to-Price Ratio**: Total stacked value ÷ Ask price → Target minimum 10:1 for webinar/VSL offers, 5:1 for high-ticket application funnels
- **Believability audit**: Flag any individual value claim the ICA would find implausible; recommend recalibration
- **Price anchor sequence**: Recommend the presenter's verbal build (the order and pacing for revealing each element on the Stack Slide)

## Step 7: Deliver Final Output Package

Produce the following named deliverables:

1. **Offer Architecture Brief** — single-page strategic summary: ICA, transformation delta, core mechanism, pricing rationale, value-to-price ratio
2. **Bonus Specification Sheet** — for each bonus: name, description (50–75 words), standalone value with valuation method, strategic function, delivery format and timeline
3. **Guarantee Script** — presentation-ready language for the guarantee, including verbal framing, conditions, and objection pre-emption lines
4. **Stack Slide Copy** — formatted cumulative Stack Slide text ready for slide design or sales page integration
5. **Presenter Talk Track** — scripted narration for walking through the stack (Brunson reveal-and-anchor cadence): what to say as each element appears, where to pause, where to re-anchor the total value
6. **Offer Integrity Scorecard**:
