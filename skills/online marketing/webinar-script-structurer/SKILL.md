## Step 2: Get Input

Ask the user to provide or confirm the following:

- **Offer type**: What are you selling at the end of this webinar? (course, coaching programme, SaaS, membership, high-ticket service, hybrid)
- **Price point and model**: One-time, payment plan, application-based — and the specific dollar amount
- **ICA profile**: Who is in the virtual room? Describe their current situation, primary pain, and the transformation they want
- **Webinar format**: Live, evergreen, hybrid (live with replay)? Target duration (60, 75, or 90 minutes)?
- **Content pillars**: Do you already have 3 core teaching points, or do you need help deriving them from your offer?
- **Known objections**: What are the top 3–5 reasons your ICA hesitates to buy?
- **Existing assets**: Any previous webinar scripts, slide decks, launch data, or testimonials to build from?
- **Delivery platform**: Where will this air? (Zoom, WebinarJam, EverWebinar, StreamYard, etc.) — this affects CTA mechanics

If FutureProof context already contains answers to any of these, confirm them with the user rather than re-asking.

## Step 3: Do the Work — Framework Selection & Synthesis

### 3a: Select Primary Architecture

Based on offer type, price point, and ICA sophistication, select and weight the structural framework blend:

| Framework | Best Fit | Core Sequence |
|---|---|---|
| **Brunson — Perfect Webinar** | Courses, memberships, offers ≤ $2,000 | Origin Story → 3 Secrets (each destroying a false belief) → Stack → Close |
| **Porterfield — Possibility-Path-Promotion** | Digital courses, warm audiences, creator-led brands | Possibility (paint the vision) → Path (teach the how) → Promotion (bridge to offer) |
| **Boyd — Sell From Stage Structure™** | High-ticket ($3,000+), coaching, application funnels | Authority Open → Problem Agitation → Framework Teach → Irresistible Offer Architecture → Urgency Close |

Most webinars benefit from a **hybrid architecture**. Map the user's inputs to a primary framework (60% weighting) and a secondary framework (40% weighting), then synthesise a unified run-of-show.

### 3b: Construct the Run-of-Show Backbone

Build the structural outline across these mandatory segments, assigning precise time blocks based on total duration:

**Segment 1 — Pre-Show & Social Proof Runway (0:00–0:05)**
- Purpose: Build energy, establish chat engagement, surface early social proof
- Tactics: Welcome arrivals by name, run a poll, tease the "big reveal" coming later
- Transition cue: "We're going to get started in 60 seconds — drop [keyword] in the chat if you're ready"

**Segment 2 — Opening Hook & Promise (0:05–0:12)**
- Purpose: Pattern-interrupt, state the singular Big Promise, establish credibility fast
- Structure: Hook statement (contrarian or curiosity-driven) → Big Promise → Credibility markers (results, logos, media) → Road map of what they'll learn
- Brunson element: "One Thing" positioning
- Boyd element: Authority Open
- Transition cue: Bridge from promise to origin story — "Let me tell you how I discovered this…"

**Segment 3 — Origin Story / Epiphany Bridge (0:12–0:22)**
- Purpose: Build emotional connection, destroy the "you're different from me" belief, pre-frame the methodology
- Structure: Before state (relatable struggle) → Epiphany moment → After state → "And today I'm going to show you exactly how"
- Brunson element: Epiphany Bridge narrative
- Porterfield element: Possibility painting through personal proof
- Key craft notes: Calibrate vulnerability to ICA expectations — corporate ICA requires restrained storytelling; creator ICA permits raw narrative
- Transition cue: "So here's what I've learned works — and I've broken it into 3 [secrets/pillars/shifts]…"

**Segment 4 — Content Pillar 1 / Secret 1 (0:22–0:35)**
- Purpose: Deliver genuine teaching value while destroying False Belief #1
- Structure: State the secret → Teach the concept → Provide a quick-win framework or example → Destroy the false belief explicitly → Mini social proof (testimonial or case study)
- Brunson element: Secret 1 — typically addresses the "vehicle" false belief (they think they need a different solution)
- Porterfield element: Path teaching — first milestone
- Transition cue: "Now that you understand [Pillar 1], the next question is…"

**Segment 5 — Content Pillar 2 / Secret 2 (0:35–0:47)**
- Purpose: Deliver teaching value while destroying False Belief #2
- Structure: Same architecture as Pillar 1
- Brunson element: Secret 2 — typically addresses the "internal" false belief (they think they can't do it)
- Porterfield element: Path teaching — second milestone
- Transition cue: "So now you have [Pillar 1] and [Pillar 2] — but there's one more piece most people miss…"

**Segment 6 — Content Pillar 3 / Secret 3 (0:47–0:58)**
- Purpose: Deliver teaching value while destroying False Belief #3 and bridging to the offer
- Structure: Same architecture as Pillar 1, but the false belief destroyed here directly creates demand for the paid offer
- Brunson element: Secret 3 — typically addresses the "external" false belief (they think external circumstances prevent success)
- Boyd element: Framework Teach — the "system" reveal that positions the offer as the complete implementation path
- Porterfield element: Promotion bridge begins here
- Transition cue: "Now, you can absolutely take these 3 [secrets/pillars] and run with them on your own. But if you want the fastest path, let me show you how I help people implement all of this…"

**Segment 7 — The Pitch Transition & Stack (0:58–1:10)**
- Purpose: Transition from teacher to offer-maker without losing trust; present the offer using value-stacking methodology
- Structure:
  - Permission bridge: "Is it okay if I share how we can work together?"
  - Restate the Big Promise
  - Introduce the offer by name
  - Stack the components (each module/deliverable presented with its standalone value)
  - Bonus stack (time-sensitive bonuses presented individually)
  - Total value vs. investment price — the "gap" reveal
  - Guarantee / risk reversal
- Brunson element: The Stack Slide methodology — building perceived value to 10x+ the price
- Boyd element: Irresistible Offer Architecture — value engineering the price anchor
- Price presentation: Calibrate to offer type (direct price for ≤$2K; "investment starts at" language for high-ticket; application CTA for $5K+)
- Transition cue: "So the question isn't whether this works — it's whether you're ready to start. Let me answer the questions I know you're thinking…"

**Segment 8 — Objection Handling & FAQ (1:10–1:18)**
- Purpose: Systematically neutralise the top 3–5 purchase objections identified in Step 2
- Structure: State each objection in the ICA's own language → Reframe → Provide proof (testimonial, case study, logical argument) → Redirect to CTA
- Present in descending order of frequency
- Boyd element: Pre-emptive objection sequencing
- Transition cue: "I've answered the most common questions — now let me tell you what happens when you click that button…"

**Segment 9 — Urgency & Close (1:18–1:25)**
- Purpose: Create legitimate urgency, issue the final call to action, close with future-state visualisation
- Structure:
  - Recap the 3 secrets/pillars in one sentence each
  - Restate the Big Promise
  - Urgency mechanism (bonuses expire, cohort closes, price increases — must be real)
  - Final CTA with explicit next-step instructions (click the button, book the call, complete the application)
  - Future-pacing: "Imagine 90 days from now…"
  - Closing line — memorable, brand-aligned sign-off
- Porterfield element: Promotion close — vision-forward language
- Boyd element: Urgency Close — scarcity architecture

**Segment 10 — Post-Close Q&A / Soft Selling (1:25–1:30)**
- Purpose: Catch late converters, answer live questions, repeat CTA naturally within answers
- Structure: Take 3–5 live questions, weave testimonials into answers, restate CTA after each answer
- Note: For evergreen webinars, this segment is replaced by a pre-recorded FAQ montage with CTA overlay

### 3c: Map Transitions & Emotional Arc

For each segment boundary, define:
- **Emotional state entering** → **Emotional state exiting**
- **Specific transition sentence** (written out, not summarised)
- **Slide cue or visual shift** (if applicable)

Plot the full emotional arc on a tension curve: Curiosity → Connection → Insight → Insight → Insight → Desire → Confidence → Urgency → Action.

### 3d: Calibrate to Offer Type

Apply offer-specific modifications:

- **Low-ticket (< $500)**: Compress pitch to 8 minutes, emphasise ease and speed, direct checkout CTA
- **Mid-ticket ($500–$2,000)**: Full stack methodology, payment plan emphasis, bonus urgency
- **High-ticket ($3,000+)**: Application/call CTA, consultative close language, extended social proof, Boyd framework weighted higher
- **SaaS / free trial**: Replace stack with feature-benefit walkthrough, demo segment replaces Pillar 3, CTA is "start free trial"
- **Membership / recurring**: Emphasise community proof, first-month-free urgency, cancel-anytime risk reversal

### 3e: Apply FutureProof Context

Cross-reference all structural decisions against:
- User's `instructions` (voice, tone, banned phrases, brand guidelines)
- Prior `experiments` (what has been tested before, what converted, what didn't)
- `recent_sessions` (any ICA refinements, offer changes, or positioning shifts)

## Step 4: Deliver Output

Produce a **Webinar Script Structure Document** containing:

### Document 1: Run-of-Show Blueprint
A timing-precise table with columns:
| Segment | Time Block | Duration | Purpose | Framework Source | Key Deliverable | Emotional State | Transition Sentence |

All 10 segments populated with the user's specific content, offer language, and ICA terminology.

### Document 2: Segment-by-Segment Outline
For each of the 10 segments, provide:
- **Objective** (one sentence)
- **Talking points** (3–7 bullet points with specific language suggestions)
- **Slide direction** (what should be on screen)
- **Social proof placement** (which testimonial or data point to deploy)
- **Transition script** (exact sentences to move to next segment)
- **Timing guardrail** (what to cut if running long, what to expand if running short)

### Document 3: False Belief Destruction Map
A 3-column table mapping:
| False Belief | Content Pillar That Destroys It | Bridge to Offer |

### Document 4: Stack Slide Architecture
The full value stack in presentation order:
- Component name → Standalone value → Description (one line)
- Bonus 1–3 with urgency triggers
- Total value, investment price, and savings calculation
- Guarantee language (verbatim draft)

### Document 5: Objection Handling Scripts
For each of the top 3–5 objections:
- Objection (stated in ICA language)
- Reframe (2–3 sentences)
- Proof point
- Redirect to CTA (one sentence)

### Delivery Format
All five documents delivered as a single structured output, clearly sectioned with headers. If the user requests it, provide a Notion-ready, Google Docs-ready, or teleprompter-formatted version.
